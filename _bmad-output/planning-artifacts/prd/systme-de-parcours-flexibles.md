# Système de Parcours Flexibles

## Le Problème

Le parcours 5 étapes est pour le client "création from scratch". Mais il existe d'autres profils :

| Type de client | Besoin | Parcours |
|----------------|--------|----------|
| Création complète | Tout construire | 5 étapes complètes |
| Optimisation | Automatiser une procédure | Étape technique uniquement |
| Modernisation | Refaire son site | Identité + Cahier des charges |
| Outil direct | Besoin clair et précis | Cahier des charges direct |

## Architecture à 3 Couches

```
COUCHE 1 : BRIEF INITIAL (Systématique)
────────────────────────────────────────
• Visio MiKL + Client
• Transcription auto
• Orpheus analyse → Brief Structuré + Profil Communication
• Output : Qui est le client + Quel est le besoin + Comment lui parler

                    │
                    ▼

COUCHE 2 : CLASSIFICATION MiKL (Décision humaine)
────────────────────────────────────────────────
MiKL décide du PARCOURS :

[A] Parcours Complet    → 5 étapes (Vision → Graduation)
[B] Parcours Partiel    → Étapes sélectionnées
[C] Parcours Ponctuel   → 1 problématique = 1 livrable
[D] Direct One          → Pas de Lab, config directe

                    │
                    ▼

COUCHE 3 : ÉLIO LAB CONFIGURÉ
─────────────────────────────
Élio reçoit :
• Le Brief Initial (contexte client)
• Le Parcours assigné (quelles étapes)
• Le Profil Communication (ton adapté)
• Les questions spécifiques à poser

Élio s'adapte au parcours, pas l'inverse.
```

## Types de Parcours

### [A] Parcours Complet — "Création Business"

5 étapes : Vision → Positionnement → Offre → Identité → Graduation

**Pour :** Client qui part de zéro

### [B] Parcours Partiel — "Accélération"

MiKL coche les étapes pertinentes, les autres sont skippées.

**Pour :** Client qui a déjà certains éléments validés

### [C] Parcours Ponctuel — "Mission"

Une seule problématique = un seul livrable (Brief Technique spécifique)

**Pour :** Client avec un besoin précis (automatisation, intégration, etc.)

### [D] Direct One — "Client Mûr"

Pas de Lab. Configuration directe de One.

**Pour :** Client qui sait exactement ce qu'il veut
