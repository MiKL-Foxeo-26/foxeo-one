# Success Criteria

## User Success — Parcours Lab en 5 Étapes

Le client Lab "réussit" quand il complète son parcours et obtient ses livrables validés.

| Étape | Nom | Livrable | Validation MiKL | Agent BMAD |
|-------|-----|----------|-----------------|------------|
| 1 | **Vision** | Brief Vision | "Je comprends ton projet" | Analyst |
| 2 | **Positionnement** | Brief Positionnement | "Tu sais à qui tu parles" | PM |
| 3 | **Offre** | Brief Offre | "Ton offre est vendable" | PM |
| 4 | **Identité** | Brief Identité | "Ton image est cohérente" | UX Designer |
| 5 | **Graduation** | Cahier des charges One | "On peut construire ton outil" | Architect |

**Critères mesurables :**

| Critère | Indicateur |
|---------|------------|
| Parcours complété | Toutes les étapes assignées validées via Validation Hub |
| Livrables produits | Briefs correspondant au parcours du client |
| Graduation | Migration vers One avec mémoire Lab conservée |
| Autonomie sur One | Client utilise son dashboard sans support constant |

## Business Success

Approche pragmatique — pas de KPIs artificiels.

| Critère | Indicateur |
|---------|------------|
| Modèle viable | Les clients Lab graduent vers One |
| Récurrence | Les clients One restent abonnés |
| Évolutions demandées | Les clients One demandent des évolutions via Validation Hub |

## Technical Success

| Critère | Indicateur |
|---------|------------|
| Opérationnel | Produit utilisable en conditions réelles (test cliente) |
| Stabilité core | Chat, Visio, Documents fonctionnent sans perte de données |
| Itérable | Feedback terrain intégrable rapidement |
| Communication agents | Orpheus et Élio communiquent via format partagé |
