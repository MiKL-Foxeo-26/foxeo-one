## 1. QUI EST FOXEO

### Identité de Marque

| Élément | Valeur |
|---|---|
| **Marque** | Foxeo |
| **Baseline** | **"Soyez rusé, pensez Foxeo."** |
| **Signature** | *Foxeo — Stratégie digitale & accompagnement de projets* |

| **Méthode** | Les 3C : Comprendre (+ Former). Créer. Concrétiser. |

### Vision

Foxeo accompagne les entrepreneurs, associations et TPE/PME dans leur transformation numérique. Pas un simple prestataire web — un **partenaire de croissance** qui reste après la livraison.

### Le Modèle Centaure

Le coeur de Foxeo repose sur le **modèle Centaure** : la combinaison d'une **intelligence artificielle disponible 24h/24** et d'un **expert humain** (MiKL) qui valide, ajuste et personnalise chaque décision.

> "L'IA fait le travail répétitif, l'humain apporte le jugement. Le client bénéficie des deux sans payer le prix des deux."

**Ce que ça change concrètement :**
- Le client n'attend jamais : Élio (l'IA) répond instantanément
- Le client n'est jamais seul face à une machine : MiKL supervise tout
- Le coût reste accessible : l'IA absorbe 80% du volume de travail
- La qualité reste premium : l'humain valide 100% des décisions critiques

### Positionnement

| Ce que Foxeo N'EST PAS | Ce que Foxeo EST |
|---|---|
| Une agence web classique | Un accompagnement IA + Humain |
| Un freelance qui livre et disparaît | Un partenaire avec suivi permanent |
| Un outil SaaS générique | Un dashboard personnalisé par client |
| Un chatbot gadget | Un assistant IA qui connaît votre projet |

### Le Profil MiKL — Développeur & Product Builder

MiKL n'est pas un consultant no-code ou un intégrateur d'outils existants. C'est un **développeur full-stack** et **product builder** qui conçoit et développe réellement les produits Foxeo.

| Compétence | Détail |
|---|---|
| **Développement full-stack** | Next.js, React, TypeScript, Supabase, PostgreSQL — MiKL code les dashboards lui-même |
| **Product building** | De l'idée au produit déployé : architecture, design, développement, DevOps |
| **Automatisation de process** | Compétence clé de l'offre — MiKL identifie les tâches répétitives du client et les automatise via des modules sur mesure |
| **IA-augmenté (Cursor/BMAD)** | Utilise l'IA comme multiplicateur de productivité pour développer à la vitesse d'une équipe |

**Ce que ça change pour le client :**
- Le produit est développé **sur mesure**, pas assemblé à partir de briques no-code
- L'automatisation de process est au coeur de la proposition de valeur : chaque workflow répétitif du client peut devenir un module
- MiKL comprend profondément la technique ET le métier du client — il traduit les besoins en solutions réelles
- Le coût reste accessible grâce à l'IA qui absorbe le volume de développement

## 4. ORPHEUS — LE CERVEAU FOXEO

### Rôle

Orpheus est le **cerveau invisible** de Foxeo. Il travaille en coulisses, dans l'environnement de développement (Cursor/BMAD), et ne communique **jamais directement** avec les clients.

### Ce qu'Orpheus fait

| Fonction | Détail |
|---|---|
| **Analyse des visios** | Lit les transcriptions Deepgram et génère des briefs structurés |
| **Détection de profil** | Identifie le profil de communication du client (analytique, expressif, pragmatique...) |
| **Configuration Élio** | Génère les fichiers de configuration personnalisés pour chaque Élio |
| **Estimation projet** | Crée des documents d'estimation (temps, coût, complexité) |
| **Documentation technique** | Génère la doc pour Élio One (guides modules, FAQ personnalisées) |
| **Retraitement brainstorming** | Transforme les outputs de sessions créatives en livrables clients |
| **Apprentissage domaine** | Apprend les patterns de pricing, timelines et méthodologies |

### Flux de communication

```
Orpheus (coulisses)
    │
    ▼ génère des documents
Élio Hub (dashboard MiKL)
    │
    ▼ MiKL valide
Élio Lab / Élio One / Élio One+ (dashboards clients)
    │
    ▼ le client reçoit
```

> Orpheus ne parle jamais au client. Il alimente Élio, qui lui parle au client. MiKL valide tout entre les deux.

## 7. STACK TECHNIQUE

### Frontend
- **Next.js 16** (App Router) + **React 19** + **TypeScript**
- **Tailwind CSS 4** + **Radix UI** + **shadcn/ui**
- **Turborepo** (monorepo : apps + packages partagés)

### Backend & Base de données
- **Supabase** (PostgreSQL, Auth, Storage, Realtime)
- **Row-Level Security (RLS)** pour l'isolation multi-tenant
- **Zod** pour la validation des schémas

### Intelligence Artificielle
- **Élio (3 agents)** : DeepSeek V3.2 (~0,28$/1M tokens input)
- **Orpheus** : Claude (via abonnement Cursor/BMAD)
- **Transcription** : Deepgram (~0,63€/heure avec diarisation)

### Services intégrés (auto-hébergés)
- **OpenVidu 3.5+** : Visioconférence WebRTC
- **Cal.com** : Prise de rendez-vous

### Services SaaS
- **Pennylane** : Facturation, devis, abonnements, comptabilité (API v2). Conformité facturation électronique sept. 2026 native. Synchronisation par polling (Edge Function cron 5min).

### Paiements & conformité
- **Stripe** connecté à Pennylane (paiements CB) + virement IBAN (Compte Pro Pennylane) + SEPA optionnel
- **Facturation électronique** : Conformité loi française (sept. 2026) — gérée nativement par Pennylane
- **RGPD** : Données hébergées en France

### Hébergement
- **Hub** : 1 instance Vercel + 1 projet Supabase (Foxeo)
- **Lab** : 1 instance Vercel + 1 projet Supabase partagé (multi-tenant Foxeo)
- **One** : 1 instance Vercel + 1 projet Supabase **par client** (propriété client)
- **Services** : VPS avec Docker Compose (OpenVidu, Cal.com)
- **Coût par client One** : ~5-7€/mois sur tiers gratuits

## 9. LES 25 PRINCIPES FONDATEURS

### Nature du produit (1-4)
1. **Le temps est la monnaie ultime** — Foxeo fait gagner du temps au client ET à MiKL
2. **Simplification cognitive** — Réduire la charge mentale, pas l'ajouter
3. **Consolidation économique** — Un écosystème qui remplace 10 outils
4. **Assistant vs Outil** — Élio est un assistant qui guide, pas un outil qu'on subit

### Architecture & modèle (5-9)
5. **LEGO model** — Template + personnalisation = chaque dashboard est unique
6. **Le chatbot comme pont relationnel** — Élio maintient le lien entre les visios
7. **L'assistant remplace la documentation** — Pas de PDF de 50 pages, Élio explique
8. **Cycle vertueux BMAD** — Chaque projet améliore le suivant
9. **Le dashboard comme miroir** — MiKL utilise son propre produit (Hub)

### Workflow client (10-14)
10. **Analyse à double niveau** — Élio traite le quotidien, Orpheus les décisions structurantes
11. **L'humain au centre** — L'IA ne décide jamais, elle propose. MiKL et le client décident
12. **Transcription = auto-documentation** — Les visios deviennent des documents exploitables
13. **Ping-pong structuré** — Brief → Validation → Ajustement → Validation, un rythme clair
14. **Déploiement à distance** — Tout se fait en ligne, pas de déplacement nécessaire

### Modèle économique (15-20)
15. **Triple source de revenus** — Labs + Setup + Abonnement + Évolutions
16. **Justifié par le chatbot actif** — L'abonnement se justifie par la présence permanente d'Élio
17. **Workflow d'évolution visible** — Le client voit que son projet peut grandir
18. **Lock-in positif & propriété client** — Le client reste parce que c'est utile, pas parce qu'il est piégé. Son outil lui appartient, il peut partir avec.
19. **Écosystème front + back** — Le client voit One, MiKL voit Hub, les deux sont connectés
20. **Hub d'intégration** — Foxeo connecte les outils existants du client

### Parcours client (21-25)
21. **Deux portes d'entrée** — Lab (création) ou One (direct), adapté à chaque situation
22. **La graduation comme moment mémorable** — Passage Lab → One = fierté du travail accompli
23. **Mémoire persistante** — Élio One se souvient de tout le parcours Lab
24. **Transparence en temps réel** — Le client voit l'avancement, pas de boîte noire
25. **Micro-ajustements continus** — Pas de gros chantiers, des améliorations progressives

## 12. SÉCURITÉ & CONFORMITÉ

- **Déploiement dual** : Lab multi-tenant (RLS) + One instance dédiée par client (propriété client)
- **Propriété client** : Le client One possède son code, ses données et sa documentation
- **Communication inter-instances** : Hub↔One via API REST signées (HMAC SHA-256)
- **Authentification** : Supabase Auth avec tokens (access 1h, refresh 30j, inactivité 8h)
- **2FA** : Obligatoire pour MiKL, optionnel pour les clients
- **Audit trail** : Actions critiques journalisées
- **Backups** : Quotidiens automatiques + hebdomadaires cold backup
- **Surveillance usage** : Monitoring capacité par instance (seuils 60/80/95%)
- **RGPD** : Données stockées en France, consentement explicite, export/suppression possibles
- **Facturation électronique** : Conformité loi française septembre 2026 via PDP certifié
- **Rate limiting** : Par utilisateur, calibré selon le plan
