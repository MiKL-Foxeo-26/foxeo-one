# Guide Utilisateur — Validation Hub

## À propos du module

Le **Validation Hub** est le centre de traitement des demandes de validation pour l'opérateur MiKL. Il regroupe toutes les soumissions de vos clients (briefs Lab et évolutions One) en un seul endroit, avec des filtres et une vue claire des priorités.

## Accès au module

Le Validation Hub est accessible depuis la sidebar du Hub Foxeo, sous l'icône "check-circle-2". Il est réservé à l'opérateur (MiKL).

## File d'attente des demandes

### Vue d'ensemble

La page principale affiche **toutes les demandes** de validation classées par statut puis par date de soumission :

1. **En attente (pending)** — demandes non encore traitées, affichées en premier
2. **Autres statuts** — approuvées, refusées, précisions demandées

### Informations affichées par demande

Pour chaque demande, vous voyez :
- **Avatar + nom + entreprise** du client
- **Type de demande** : Badge terracotta pour les Briefs Lab, orange pour les Évolutions One
- **Titre** de la demande
- **Date de soumission** en format relatif ("il y a 2h", "hier")
- **Statut** : badge coloré selon l'état (jaune=en attente, vert=approuvé, rouge=refusé, bleu=précisions)

## Filtres disponibles

- **Statut** : Tous / En attente / Approuvé / Refusé / Précisions demandées
- **Type** : Tous / Brief Lab / Évolution One
- **Tri** : Par date (plus ancienne ou plus récente)

## Navigation vers le détail

Cliquez sur une demande pour accéder à sa **vue détaillée** (Story 7.2).

## Compteur de demandes en attente

Le header affiche un **badge jaune** avec le nombre de demandes en attente lorsqu'il y en a. Ce compteur est aussi disponible dans la navigation sidebar (Story 7.6 — temps réel).

## État vide

Si aucune demande n'est en attente, un message "Aucune demande en attente — tout est à jour !" s'affiche avec une icône verte.
